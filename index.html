Suggested Priority Rubric

The suggested priorities (T0–T3) were derived by scoring each source system across three dimensions:

Formula: Blast Radius × Ingestion Fragility × Criticality Flags in Code = Suggested Tier

Tier Blast Radius Ingestion Fragility Code Signals Sources
T0 Very High. 30+ downstream DAGs depend on a single dimension (Salesforce). Amazon Connect CTR feeds routing, surveys, and SLA metrics — any delay cascades across nearly every support, fraud, and QA pipeline. High. Both flow through Fivetran with no formal upstream SLA. A sync delay or schema drift has no fallback. criticality: high, EDW_OPEX_DEFAULT_DAG_ARGS_CRITICAL, priority_weight: 200 Salesforce, Amazon Connect
T1 High. Checkr has 10 sub-report DAGs downstream. Sprout Social spans 8 pipelines. Stripe Radar feeds fraud scoring. High. Sprout Social and Persona use direct REST API calls via PythonOperator with no circuit breaker or fallback cache. An API outage, rate limit, or token expiry causes full DAG failure with no partial recovery. priority_weight: 200 (Checkr). Default Tier 1 for others, but operationally fragile due to raw API integration pattern. Checkr, Stripe Radar, Sprout Social, Persona
T2 Moderate. 2–4 pipelines each. Observe AI’s unified QA aggregation has 9 upstream sensors but is self-contained within the QA domain. Low. All use Snowflake data shares — inherently stable (no auth tokens, no rate limits, no API calls). Risk is limited to vendor-side data freshness. Default Tier 1 with no elevated criticality flags. No explicit SLA or priority overrides in code. Observe AI, MaestroQA, Cornerstone
T3 Low. 1–3 pipelines each. Downstream impact is narrow or domain-specific. Low. Google Sheets are manually maintained (delays are human-driven, not systemic). MaxMind runs weekly and the cadence is sufficient. Arize AI is explicitly a temporary solution pending a TPDI connector. Default Tier 1. Google Sheets CX tables have criticality: high but the source is manual/human-owned, limiting systemic risk. Google Sheets, MaxMind, Arize AI
